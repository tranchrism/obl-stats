# Oakland Hockey Stats

A local, mobile-friendly hockey stats dashboard for the Oakland Sharks Ice TimeToScore league pages.

Stats are intentionally limited to 2017 onward.

## What It Uses

The Oakland public stats page does not appear to expose a documented public API. The public pages do expose consistent HTML endpoints for:

- seasons and standings
- division player and goalie leaders
- league and team schedules
- team player, goalie, situation, and special-teams stats
- scoresheet PDF links for completed games

The `oss-scoresheet` page also bootstraps an internal signed API at `api.sharksice.timetoscore.com` for the game-center widget. That API is real, but it is not documented and the first version of this app uses the stable public pages instead.

## Run

```bash
python3 server.py
```

Then open:

```text
http://127.0.0.1:8765
```

## Public Hosting

The recommended free hosting path is GitHub Pages with static JSON generated by `export_static.py`.

Why this shape:

- GitHub Pages serves the frontend and JSON files for free.
- GitHub Actions refreshes the static data once per day.
- Player profiles load quickly because profile JSON is generated ahead of time instead of scraped on demand.
- No public server, database, or paid cache layer is required for the MVP.

Build the static site locally:

```bash
python3 export_static.py --out dist
```

The included GitHub Actions workflow deploys `dist/` to GitHub Pages:

- on pushes to `main`
- manually via `workflow_dispatch`
- nightly at `10:30 UTC`, roughly middle-of-the-night Pacific time

For local backend development, keep using `python3 server.py`. For GitHub Pages, the frontend automatically reads from `data/*.json` instead of `/api/*`.

## Local API

- `GET /api/seasons`
- `GET /api/standings?season=0`
- `GET /api/division-stats?season=76&level=99&conf=0`
- `GET /api/schedule?season=76`
- `GET /api/team?season=76&team=718`
- `GET /api/player?name=Christopher%20M%20Tran&season_type=regular`
